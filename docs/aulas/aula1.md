# 1. Bem-vindo(a)! O que é Back-End?

Quando você usa um app, faz login, pesquisa algo, envia uma mensagem ou conclui uma compra online, existe uma parte do sistema que funciona “por trás das cortinas”.  
É o **back-end** — a parte que o usuário **não vê**, mas que faz tudo acontecer.

Enquanto o **front-end** é o que aparece na tela, o **back-end** é onde ficam:

- A lógica do sistema  
- O processamento de dados  
- A comunicação com o banco de dados  
- As regras de negócio  
- A segurança (autenticação e autorização)

Pense assim:

> **Front-end:** entrega a pizza 🍕  
> **Back-end:** cozinha, ingredientes, pedidos, caixa, produção 👨‍🍳

# 2. Linha do Tempo do Back-End

## Anos 1960–1970 — O começo
- Sistemas centralizados em mainframes  
- Aplicações monolíticas  
- Nada de aplicações web  

## Anos 1990 — A internet surge
- Sites dinâmicos ficam populares  
- PHP, CGI e Perl crescem  
- Modelo Cliente → Servidor → Banco de Dados se torna padrão  

## 2010 em diante — A era moderna
- Nuvem, APIs, microserviços, contêineres  
- Node.js unifica JavaScript no front e back  
- Sistemas escaláveis e modulares

---

# 3. O que o Back-End faz?

O back-end é responsável por:

- Processar informações  
- Validar dados  
- Salvar/buscar conteúdo no banco  
- Garantir segurança  
- Integrar sistemas  
- Aplicar regras de negócio  

Exemplos reais:
- Cadastro e login  
- Histórico de pedidos  
- Recomendação de produtos  
- Atualização automática de estoque  

Sem back-end, nada funciona.

---

# 4. Front-End x Back-End

| O que é? | Front-End | Back-End |
|---------|-----------|-----------|
| Área | Interface com o usuário | Processamento e lógica |
| Linguagens comuns | HTML, CSS, JS | Python, C#, Java, JavaScript (Node) |
| Foco | Visual | Dados, segurança, performance |
| Usuário vê? | Sim | Não |

# 6. Conceitos Essenciais

### Servidor  
Computador que recebe e responde requisições.

### Requisição  
Pedido feito pelo usuário (ex.: fazer login).

### Resposta  
Retorno enviado pelo servidor.

### Banco de Dados  
Onde os dados ficam armazenados.

### API  
Rota usada para comunicação entre sistemas.

**Requisição → Processamento → Resposta**  
Esse é o ciclo básico do back-end.

---

# 7. Comunicação Front-End ↔ Back-End

Fluxo simplificado:
Usuário → envia um pedido
Servidor → processa
Servidor → devolve a resposta
Usuário → vê o resultado


Exemplo: pesquisa de produtos.
